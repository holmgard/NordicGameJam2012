» Examples Overview

» Flash Examples:

  • SFS2X-Connector: this is the most simple SFS2X application possible. It just creates the SmartFox object, load the external configuration and connect to the server

  • SFS2X-SimpleChat: a simple chat application that demonstrates the fundamental requests such as login, joining rooms, sending chat messages, creating a new Room, disconnecting.

  • SFS2X-AdvancedChat: the application extends the previous chat by introducing the Room Groups. In this example you will be watching the events of two different Groups called default and chat. The application also demonstrates the new Room commands for changing name, password and size of a Room at runtime. Private chatting is also included.

  • SFS2X-Tris: this demo shows the process to build a complete turn-based game with server side logic. Thanks to the free SmartFoxBits2 UI components distributed in this release you will be able to build the Lobby in a snap and reduce the boilerplate code by a large degree. All the common UI logic (room list, user list, chatting) will be handled by the components while you can concentrate on the game itself. The server side source code is also included.

  • SFS2X-Lobby: this example showcases some of the new Game API features. You can create public games that require Users to have certain prerequisistes to join the game. Or you can create a private Game and at the same time invite other online friends to join and play.

    This is an example of how you can check the Public Games:
    • Once logged in choose your country and ranking from the My Player Details panel
    • Now create a new public Game Room. The Room will be configured to accept Users from the same country as yours and with a raking greater or equal to yours
    • Now launch another client and try joining the Room, you will see that the join will be refused if the User doesn't match the Room settings
    • Also notice that the Game started/stopped status is kept updated in the Lobby for the other Users outside the Game

    This is an example of how you can check the Private Games:
    • Start 3 instances of the example with three different Users, say Kermit, Piggy and Gonzo
    • Let Gonzo create a private game and select Kermit and Piggy as invited players in the Game
    • When you create the game both Kermit and Piggy will be notified of the invitation. If they will accept the invitation on time they will be joined and the Game will start.

    • You can learn more details abou the Game API in the specific article provided with this documentation.
» Unity3D Examples:
  • Unity Connector: this is the most simple SFS2X application possible. It just creates the SmartFox object and connect to the server showing the result.

  • Unity SimpleLobby: a simple chat application that demonstrates the fundamental requests such as login, joining rooms, sending chat messages, creating a new Room, disconnecting.

  • Unity SFS2XTris: this demo shows the process to build a complete turn-based game with server side logic. It is the equivalent of the SFS2XTris done in Flash only with a much cooler 3D game board! You can start two clients and play a match, you could also compete against a Flash client. The example talks to the the same server Extension used in the Flash example.

» Importing Flex-based examples in Flex Builder

With the exclusion of the Connector example all other examples were mostly written in Flash Builder 4 using the Flex Framework 3.5 which ensure compatibility with the previouse Flex Builder 3

The Flex projects were exported as zipped archives. This is how you can import them in Flex/Flash Builder:

  • From the main menu select File » Import » Flex Project...
  • In the dialogue box choose Archive File and browse to the location of the .zip file you want to import
  • Click Finish and you are done.

The Connector example requires Flash CS4 or higher.
